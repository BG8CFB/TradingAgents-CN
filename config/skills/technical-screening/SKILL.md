# 技术面系统化选股技能

## 适用场景

- 用户要求"从技术面筛一批股票"
- 分析师需要识别 K 线形态（头肩底、双底、突破）
- 需要计算技术指标（MACD、RSI、KDJ、布林带）

## 选股信号体系

### 趋势类信号

| 信号 | 判定 | 强度 |
|---|---|---|
| 均线多头排列 | MA5 > MA10 > MA20 > MA60 | ★★★ |
| MACD 金叉 | DIF 上穿 DEA | ★★ |
| 突破平台 | 收盘价 > 20日内最高价 且放量 | ★★★ |

### 反转类信号

| 信号 | 判定 | 强度 |
|---|---|---|
| RSI 超卖反弹 | RSI < 30 后回升至 30 上方 | ★★ |
| KDJ 金叉 | K 上穿 D 且 J < 50 | ★★ |
| 缩量回踩支撑 | 跌破 MA20 后快速收回 且缩量 | ★★★ |

### 风险类信号

| 信号 | 判定 | 强度 |
|---|---|---|
| 顶背离 | 价格新高但 MACD 未新高 | ★★★（看跌） |
| 量价背离 | 价格上涨但成交量萎缩 | ★★（看跌） |
| 均线死叉 | MA5 下穿 MA20 | ★★（看跌） |

## 脚本化能力

本技能提供脚本入口 `technical-screening.calc_indicators`，LLM 可直接调用计算技术指标，无需手动实现公式。

脚本依赖：
- `mplfinance`（用于 K 线图渲染，可选）
- Python 标准库（pandas/numpy 已被项目依赖）

依赖未安装时脚本会自动降级：返回指标数值但不渲染图表。

## 选股流程

### Step 1：拉取数据

调用 `daily_quotes` 获取至少 120 个交易日数据。

### Step 2：计算指标

调用 `technical-screening.calc_indicators` 获取：
- MA5 / MA10 / MA20 / MA60
- MACD (DIF, DEA, Histogram)
- RSI(6, 12, 24)
- KDJ
- 布林带（上轨、中轨、下轨）

### Step 3：信号匹配

按上述信号体系扫描最近 5 个交易日，记录触发信号。

### Step 4：综合评分

```
技术分 = Σ(信号强度 × 方向系数)
方向系数：看涨=+1，看跌=-1
```

### Step 5：输出

| 评分 | 建议 |
|---|---|
| > 5 | 技术面强势，关注 |
| 0 ~ 5 | 中性，观望 |
| < 0 | 技术面弱势，回避 |

## 输出格式

分析报告必须包含：

1. **最近 5 日信号清单**：日期、信号名、方向、强度
2. **技术评分**：数值 + 评级
3. **关键价位**：支撑位（MA20/MA60）、阻力位（近期高点）
4. **图表**（可选）：若 mplfinance 可用，附 K 线图说明

## 常见错误

- ❌ 只看单日信号：至少看 3 日确认
- ❌ 忽略成交量：所有突破信号必须配合放量
- ❌ 在低位死叉恐慌：低位死叉常是洗盘，需结合位置判断
